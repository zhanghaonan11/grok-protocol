# 手动执行：注册 / 转换 JSON / 上传 CPA

面向需要**手动分步操作**的场景。默认在项目根目录执行：

```bash
cd /path/to/grok-protocol
source .venv/bin/activate   # 推荐；需要 curl_cffi 时必须
```

相关配置见 `config.json`（可参考 `config.example.json`）。

---

## 0. 产物与路径约定

| 产物 | 路径 / 格式 | 说明 |
| --- | --- | --- |
| 账号行 | `accounts_*.txt` / `accounts_http_*.txt` | `email----password----sso` |
| OAuth 凭证 | `xai_credentials/xai-<email>.json` | CPA / CLIProxyAPI 兼容 |
| SSO 旁路文件 | `xai_credentials/xai-<email>.sso` | 可选，与 JSON 同名 |
| 邮箱凭证 | `mail_credentials.txt` | 临时邮箱侧信息 |

配置关键字段：

```json
{
  "xai_oauth_auto": true,
  "xai_oauth_output_dir": "xai_credentials",
  "cpa_api_url": "http://192.168.1.1:8317",
  "cpa_api_key": "你的管理密钥",
  "cpa_auto_upload": true,
  "cpa_use_local_name": true,
  "cpa_skip_duplicates": true
}
```

**自动上传说明：**

- `cpa_auto_upload=true` 时，**写出/更新 JSON 的那一刻**会尝试推 CPA。
- 目录里已有存量 JSON **不会**自动同步，需要手动推送（见第 3 节）。
- 远端已有同名/重复凭证时，`cpa_skip_duplicates=true` 会跳过。

敏感文件勿提交仓库（已被 `.gitignore` 覆盖）。

## 1. 手动注册

### 1.1 最短路径（读 `config.json`）

```bash
python grok_register_ttk.py http register \
  --mail-config config.json \
  --output-dir xai_credentials \
  --accounts-output accounts_manual.txt
```

成功后通常得到：

1. `accounts_manual.txt`（账号行）
2. 若 `xai_oauth_auto=true`，同时写出 `xai_credentials/xai-*.json`
3. 若 `cpa_auto_upload=true` 且 CPA 已配置，写出 JSON 时会尝试自动推送

### 1.2 带代理 + captcha 服务

```bash
python grok_register_ttk.py http register \
  --proxy-file proxies.txt --proxy-random \
  --proxy-parent http://127.0.0.1:7890 \
  --mail-config config.json \
  --turnstile-provider capsolver \
  --turnstile-api-key "$CAPSOLVER_API_KEY" \
  --output-dir xai_credentials \
  --accounts-output accounts_manual.txt
```

常用 captcha：`capsolver` / `yescaptcha` / `2captcha`。  
也可用环境变量：`XAI_TURNSTILE_PROVIDER`、`XAI_TURNSTILE_API_KEY`。

### 1.3 全屏 TUI 批量注册

```bash
./http_tui.sh
# 或
./http_tui.sh --config config.json --count 5 --workers 2
```

- 批次日志：`http_runs/`
- 成功账号汇总：`accounts_http_*.txt`
- 快捷键：`q` 停止/退出，`↑`/`↓` 滚动日志，`l` 回到最新

### 1.4 先探测邮箱（不注册）

```bash
python grok_register_ttk.py http mail-probe --mail-config config.json
```

期望日志含：`[+] mail-probe ok email=…`

### 1.5 仅注册、不在本命令内换 OAuth（可选）

若你只想先拿账号行，再单独转换：

```bash
python grok_register_ttk.py http register \
  --mail-config config.json \
  --accounts-output accounts_manual.txt
```

然后走第 2 节转换。

## 2. 转换 JSON（SSO → xai-*.json）

输入可以是：

- 纯 SSO cookie（一行一个）
- 账号行：`email----password----sso`（脚本默认取最后一段 SSO）

### 2.1 项目封装命令

```bash
# 文件输入
python grok_register_ttk.py http credential \
  --sso-file accounts_manual.txt \
  --output-dir xai_credentials

# 单条 SSO
python grok_register_ttk.py http credential \
  --sso "eyJ...." \
  --email "user@example.com" \
  --output-dir xai_credentials
```

### 2.2 直接用 `sso_to_auth_json.py`（推荐处理账号列表）

**鉴权换票**（账号行里的 session SSO 用这个）：

```bash
python sso_to_auth_json.py \
  --mode auth \
  --sso accounts_manual.txt \
  --out-dir ./xai_credentials \
  --workers 8
```

**不鉴权仅格式转换**（输入本身已是 access_token / 完整 JSON）：

```bash
python sso_to_auth_json.py \
  --mode noauth \
  --sso tokens.txt \
  --out-dir ./xai_credentials \
  --workers 8
```

单条：

```bash
python sso_to_auth_json.py \
  --mode auth \
  --sso-cookie "eyJ...." \
  --email "user@example.com" \
  --out-dir ./xai_credentials
```

### 2.3 只补「有账号行、缺 JSON」的邮箱

```bash
python - <<'PY'
from pathlib import Path
import glob, json
import sso_to_auth_json as conv

OUT = Path("xai_credentials")
OUT.mkdir(parents=True, exist_ok=True)

json_emails = set()
for p in OUT.glob("*.json"):
    try:
        d = json.loads(p.read_text(encoding="utf-8"))
        e = str(d.get("email", "")).strip().lower()
        if e:
            json_emails.add(e)
    except Exception:
        pass

best = {}  # email -> sso
for path in sorted(set(glob.glob("accounts_*.txt") + glob.glob("accounts_http_*.txt"))):
    for line in Path(path).read_text(encoding="utf-8", errors="replace").splitlines():
        line = line.strip()
        if "----" not in line:
            continue
        parts = line.split("----")
        if len(parts) < 3:
            continue
        email = parts[0].strip().lower()
        sso = parts[-1].strip()
        if not email or not sso or email in json_emails:
            continue
        prev = best.get(email)
        if prev is None or len(sso) >= len(prev):
            best[email] = sso

print(f"待转换: {len(best)}")
ok = fail = 0
items = list(best.items())
for i, (email, sso) in enumerate(items, 1):
    print(f"[{i}/{len(items)}] {email}")
    if conv.process_auth_one(i, len(items), sso, OUT, email):
        ok += 1
    else:
        fail += 1
print(f"完成: 成功 {ok}, 失败 {fail}")
PY
```

> 需要 `.venv` 中的 `curl_cffi`。若系统 Python 报 `No module named curl_cffi`，请用：
>
> ```bash
> .venv/bin/python sso_to_auth_json.py --mode auth --sso accounts_manual.txt --out-dir ./xai_credentials
> ```

### 2.4 转换结果检查

```bash
# JSON 数量
ls xai_credentials/*.json | wc -l

# 账号列表唯一邮箱 vs JSON 邮箱
python - <<'PY'
from pathlib import Path
import glob, json, re

emails = set()
for p in glob.glob("accounts_*.txt") + glob.glob("accounts_http_*.txt"):
    for line in Path(p).read_text(encoding="utf-8", errors="replace").splitlines():
        m = re.findall(r"[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\.[A-Za-z]{2,}", line)
        emails.update(x.lower() for x in m)

json_emails = set()
for p in Path("xai_credentials").glob("*.json"):
    try:
        d = json.loads(p.read_text(encoding="utf-8"))
        e = str(d.get("email", "")).strip().lower()
        if e:
            json_emails.add(e)
    except Exception:
        pass

print("账号邮箱:", len(emails))
print("JSON 邮箱:", len(json_emails))
print("缺 JSON:", len(emails - json_emails))
for e in sorted(emails - json_emails):
    print(" ", e)
PY
```

## 3. 上传 CPA

### 3.1 配置检查

```bash
python - <<'PY'
import json
import cpa_push

cfg = json.load(open("config.json", encoding="utf-8"))
print("url:", cpa_push.normalize_cpa_base_url(cfg.get("cpa_api_url")))
print("key:", cpa_push.mask_secret(cfg.get("cpa_api_key")))
print("auto:", cfg.get("cpa_auto_upload"))
print("skip_dup:", cfg.get("cpa_skip_duplicates"))
print("use_local_name:", cfg.get("cpa_use_local_name"))
print("dir:", cfg.get("xai_oauth_output_dir", "xai_credentials"))
print(cpa_push.check_cpa_connection(cfg["cpa_api_url"], cfg["cpa_api_key"])["message"])
PY
```

### 3.2 手动全量推送本地 JSON（补传存量）

```bash
python - <<'PY'
import json
import cpa_push

cfg = json.load(open("config.json", encoding="utf-8"))
result = cpa_push.push_local_credentials(
    base_url=cfg["cpa_api_url"],
    api_key=cfg["cpa_api_key"],
    output_dir=cfg.get("xai_oauth_output_dir", "xai_credentials"),
    use_local_name=bool(cfg.get("cpa_use_local_name", True)),
    skip_duplicates=bool(cfg.get("cpa_skip_duplicates", True)),
    log=print,
)
print(result.get("message"))
print(
    f"total={result.get('total')} "
    f"success={result.get('success')} "
    f"failed={result.get('failed')} "
    f"skipped={result.get('skipped')}"
)
PY
```

强制覆盖远端同名（**会重传已存在凭证**）：

```bash
python - <<'PY'
import json
import cpa_push

cfg = json.load(open("config.json", encoding="utf-8"))
result = cpa_push.push_local_credentials(
    base_url=cfg["cpa_api_url"],
    api_key=cfg["cpa_api_key"],
    output_dir=cfg.get("xai_oauth_output_dir", "xai_credentials"),
    use_local_name=bool(cfg.get("cpa_use_local_name", True)),
    skip_duplicates=False,  # 不去重
    log=print,
)
print(result.get("message"))
PY
```

## 5. 常见问题

### Q1：为什么账号列表有 29 个，JSON 只有 17 个？

注册成功会**立刻**写 `accounts_*.txt`；OAuth JSON 还要再走 **SSO → Device Flow 换票**。  
换票失败/跳过时，就会出现「有账号行、无 JSON」。

处理：用第 2.2 / 2.3 节对缺 JSON 的账号行做 `auth` 转换。

### Q2：有 JSON 就会自动上传吗？

**不会。**  
只有在写出 JSON 时、且 `cpa_auto_upload=true`、CPA 地址和 key 有效，才会自动推。  
存量 JSON 需要第 3.2 节手动推送。

### Q3：推送显示 success=0、skipped=全部？

说明远端已有同名凭证。这是 `cpa_skip_duplicates=true` 的正常行为。  
要覆盖再推，把 `skip_duplicates=False`（见 3.2）。

### Q4：`No module named curl_cffi`

```bash
source .venv/bin/activate
# 或
.venv/bin/python sso_to_auth_json.py --mode auth --sso accounts_manual.txt --out-dir ./xai_credentials
```

### Q5：鉴权失败怎么办？

- SSO 可能已过期，无法换票
- 网络/代理异常
- 可对失败邮箱重试 1～2 次；仍失败需重新注册

### Q6：帮助入口

```bash
python grok_register_ttk.py http --help
python sso_to_auth_json.py --help
```
