# 抓包协议分析报告（脱敏版）

**数据源：** 浏览器 Recorder / CDP HTTP 事件（已脱敏）  
**场景：** CLI Proxy API 发起 xAI OAuth 授权 → accounts.x.ai 邮箱密码登录 → OAuth consent → 本地回调落盘 token  

**注意：** 原始 Recorder JSON 含 cookie / 密码 / 管理端 token，**不要提交或分享**。本文件仅保留协议结构。

| 项目 | 内容 |
|------|------|
| 场景 | CLI Proxy API / xAI OAuth Web 授权登录 |

| 本地回调 | `http://127.0.0.1:<port>/callback` |

## 2. 总体链路（按时序）

### 阶段 A：管理端发起 OAuth

1. 管理端生成 authorize URL（PKCE S256 + state + nonce）
2. 轮询授权状态 → 前期 `wait`

### 阶段 B：打开账号站

3. 加载静态资源、Turnstile、监控
4. `GET /sign-in?redirect=oauth2-provider&return_to=/oauth2/consent?...`

### 阶段 C：邮箱登录

5. `POST /auth_mgmt.AuthManagement/CheckDomainSSOEnabled`（gRPC-Web）
6. `POST /api/rpc` `createSession`（email + password + turnstileToken + 可选 castleRequestToken）
7. 成功响应：`{"cookieSetterUrl":"https://auth.grokipedia.com/set-cookie?q=<JWT>"}`

### 阶段 D：跨域 Cookie

8. 跟随 cookie setter 四跳，最终在 `.x.ai` 写入 `sso` / `sso-rw`

### 阶段 E：OAuth Consent

9. `POST /oauth2/consent` Server Action，`action=allow` → 授权 `code`
10. 用 `code + code_verifier` 换 token，写出 CLIProxyAPI 兼容 JSON

## 3. 关键协议

### createSession（登录）

```json
{
  "rpc": "createSession",
  "req": {
    "createSessionRequest": {
      "credentials": {
        "case": "emailAndPassword",
        "value": {
          "email": "<email>",
          "clearTextPassword": "<password>"
        }
      }
    },
    "turnstileToken": "<turnstile>",
    "castleRequestToken": "<optional-castle>"
  }
}
```

### 注册链路（live JS / HTTP 模式，与本登录抓包互补）

1. `CreateEmailValidationCode`（gRPC-Web，email + 可选 castle）
2. 收信 → `VerifyEmailValidationCode`（gRPC-Web，email + code）
3. Next Server Action：`createUserAndSessionRequest` + turnstile + conversionId + 可选 castle
4. 成功时 RSC 中带 `https://auth.grokipedia.com/set-cookie?q=<JWT>`（Flight typed string：`N:T<hex>,url`）
5. 再走 OAuth consent → token 文件

### OAuth consent POST 体（结构）

```json
[{
  "action": "allow",
  "clientId": "<cli-client-id>",
  "redirectUri": "http://127.0.0.1:<port>/callback",
  "scope": "openid profile email offline_access grok-cli:access api:access",
  "state": "<state>",
  "codeChallenge": "<s256>",
  "codeChallengeMethod": "S256",
  "nonce": "<nonce>",
  "principalType": "User",
  "principalId": "",
  "referrer": "cli-proxy-api"
}]
```

## 4. Cookie / 会话

- 登录前：`xai_anon_id`、`redirect`、`referrer`、`__cf_bm` 等
- 登录后：`sso` / `sso-rw`（JWT payload 含 `session_id`）、`last-logged-in-with=EMAIL`

---

## 5. 风控要点

| 信号 | 说明 |
|------|------|
| Turnstile | 密码登录 / 注册提交必填；需新鲜 token；HTTP 模式用 captcha 服务或浏览器 capture |
| Castle | 浏览器 SDK `createRequestToken`；客户端超时可空；HTTP 模式按可选字段转发 |
| 出口 IP | 求解 Turnstile 的出口尽量与后续 xAI 请求一致（粘性代理） |

---

## 6. 复现最小闭环

```text
authorize URL
  → createSession / 注册 Server Action
  → set-cookie 四跳拿 sso
  → oauth2/consent allow
  → code + PKCE → token JSON
```

无浏览器实现见 `xai_http_flow.py` 与 `USAGE.md`。
